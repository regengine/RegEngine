5. AUTH FLOW COMPLETENESS & GAPS
=================================================================

AUTH IMPLEMENTATION:
- Auth Context: /lib/auth-context.tsx
  * Manages: user, accessToken, apiKey, tenantId, adminKey
  * localStorage-backed persistence
  * isAuthenticated = !!user && !!accessToken
  * isHydrated flag to prevent SSR mismatch

- Login Flow:
  * /app/login/page.tsx
  * QA presets available (test@example.com, admin@example.com)
  * Redirects to /dashboard or /sysadmin based on is_sysadmin flag

PAGES WITHOUT AUTH GATES:

Critical Gap: Dashboard pages use layout but NO ROUTE-LEVEL AUTH

1. UNPROTECTED MARKETING/PUBLIC PAGES (Expected):
   ✓ /
   ✓ /login
   ✓ /waitlist
   ✓ /contact
   ✓ /privacy
   ✓ /docs/*
   ✓ /tools/* (notice-validator, bias-checker, etc.)

2. PARTIALLY PROTECTED - Layout only:
   /dashboard/* — All dashboard pages rely on layout.tsx
   - No per-page useAuth() checks
   - No redirectToLogin redirect
   - Could be accessible to non-authenticated users
   - Layout renders with children even if !user
   
   /owner/* — Owner dashboard pages
   - Similar layout-only protection
   - No role verification (should require is_sysadmin)

3. DEMO PAGES (Unauthenticated):
   /demo/supply-chains
   /demo/mock-recall
   /demo/recall-simulation
   - No auth checks
   - Can view detailed supply chain data without login

4. SPECIAL PAGES (Mixed):
   /app/portal/[portalId]/page.tsx
   - Supplier portal (public-ish but with portalId)
   - No auth required but portalId gating exists
   
   /app/onboarding/* — Onboarding flow
   - Not gated, but follows login flow
   
   /app/verticals/food-safety/* — Public vertical pages
   - Accessible without auth
   - Allowed via middleware (ALLOWED_VERTICALS)

MISSING PROTECTIONS:
- No ProtectedRoute wrapper component
- No middleware-based auth (only vertical/docs blocking)
- Dashboard/owner pages rely on UX only (no backend enforcement)
- No permission checks on API calls (roles not validated client-side)
