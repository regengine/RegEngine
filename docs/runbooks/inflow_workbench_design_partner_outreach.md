# Inflow Workbench Design-Partner Outreach

Purpose: convert the new Inflow Workbench loop into 3-5 concrete design-partner conversations.

## Target Profile

- Food companies, distributors, fresh produce suppliers, and co-packers with FSMA 204 exposure.
- Teams that already receive supplier spreadsheets, EDI extracts, PDFs, screenshots, or partial API feeds.
- Buyers who need to know which lots are FDA-export ready and which suppliers are blocking readiness.

## Call Opener

"I am not trying to sell a generic compliance dashboard. I want to test whether RegEngine can take one messy supplier feed, tell you what is missing, and show whether that lot could support an FDA export."

## Demo Promise

- Upload or load messy supplier data.
- Run preflight validation.
- Show missing KDEs and rule failures in plain language.
- Open the fix queue.
- Show the commit gate and readiness score.
- Explain what would be required before production evidence commit.

## Success Criteria

- Prospect shares one real or anonymized supplier feed.
- Prospect can name one supplier readiness problem RegEngine made clearer.
- Prospect agrees to a second session or LOI discussion.
