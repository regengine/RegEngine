### Pipeline

Supplier Data -> Inflow Lab -> Engine -> Commit Gate -> Evidence + Export

### Feature Copy

**Supplier Data Readiness:** Upload or simulate CSV, spreadsheet, EDI-style, API, and supplier portal records in Inflow before they touch production evidence. Saved integration profiles preserve supplier-specific field mappings so the same feed can be replayed and improved.

**Preflight Validation:** The Engine evaluates each CTE against FSMA 204 KDE requirements, lot linkage expectations, malformed values, duplicate risks, and export readiness impact. Failures generate plain-language remediation work instead of disappearing into a rejected payload.

**Commit Gates:** RegEngine keeps simulation, preflight, staging, and production evidence decisions explicit. Production evidence requires authentication, tenant-scoped persistence, provenance, and clean readiness signals before records can support export.

**Readiness Score + Export:** Readiness scores combine KDE completeness, rule outcomes, unresolved fixes, provenance, and export eligibility. Validated tenant-scoped records can then support FDA-ready sortable spreadsheets and structured export packages.

**Audit Trail + Verification:** Committed evidence is hash chained, append-only, tenant-scoped, and independently verifiable.
