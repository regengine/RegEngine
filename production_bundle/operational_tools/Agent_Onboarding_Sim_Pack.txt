# Agent Onboarding Simulation Pack
# RegEngine Fractal Agent Swarm — Validation Harness
# ══════════════════════════════════════════════════

## Purpose

This pack validates that agents are properly configured and produce
correct output. Run these steps for every new persona addition or
after modifying the swarm infrastructure.

## Pre-Flight Checks

1. Verify all persona files exist:
   $ python scripts/summon_agent.py --list

2. Verify orchestrator registry:
   $ python scripts/swarm_orchestrator.py --roster

3. Verify protocol files exist:
   $ ls -la .agent/protocols/

## Test Scenarios

### Scenario 1: Single Agent Summon
For each agent in the roster:
   $ python scripts/summon_agent.py --role {key} --task "Status report"
Expected: Prompt generated with persona + constitution + task

### Scenario 2: JSON Output Mode
   $ python scripts/summon_agent.py --role qa --task "Test run" --output json
Expected: Prompt includes output schema instructions

### Scenario 3: Agent Chain
   $ python scripts/swarm_orchestrator.py --chain fsma,security,qa --task "Test chain"
Expected: Chain execution plan displayed, first agent prompt generated

### Scenario 4: Predefined Sweep
   $ python scripts/swarm_orchestrator.py --sweep security
Expected: Security sweep chain initiated

### Scenario 5: Output Validation (Pass)
Create valid_output.json:
{
  "agent": "Bot-QA",
  "task": "Test validation",
  "timestamp": "2026-02-09T00:00:00Z",
  "status": "completed",
  "confidence": 0.95,
  "files_changed": [],
  "tests": {"added": 0, "modified": 0, "all_passing": true},
  "risks": [],
  "recommendations": [],
  "handoff": null
}
   $ python scripts/swarm_orchestrator.py --validate valid_output.json
Expected: ✅ Output from Bot-QA is valid.

### Scenario 6: Output Validation (Fail)
Create invalid_output.json:
{
  "agent": "Bot-Unknown",
  "task": "Test",
  "status": "invalid_status"
}
   $ python scripts/swarm_orchestrator.py --validate invalid_output.json
Expected: ❌ Validation FAILED with multiple errors

### Scenario 7: Workflow Execution
   Run /security_audit workflow
   Run /full_feature_pipeline workflow
   Run /onboarding_validation workflow
Expected: All workflow steps are actionable
